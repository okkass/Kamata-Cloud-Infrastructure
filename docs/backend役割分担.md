# それぞれの要素の説明

## handler

api/配下にあるファイルたち。HTTPリクエストを受け取るエントリーポイント。

### このファイルで何をするのか

- HTTPリクエストを受け取る
- リクエストのバリデーションを行う(utilsのserviceResultHandlerを利用)
- serviceを取得、実行する

### このファイルの完成状況

- 完成してる。基本的に変更不要。asyncつけたりくらい。

## service

services/配下にあるファイルたち。ビジネスロジック(人間が決めた仕様やルール)を実装する。repository、gatewayを呼び出して、データの取得や保存を行う。

### このファイルで何をするのか

- ビジネスロジックを実装する(例: リソースにアクセスする権限を持ってるかをチェックする)
- repository、gatewayを呼び出して、データの取得や保存を行う

### このファイルの完成状況

- モック向けの状態になってる。ビジネスロジックやgatewayの呼び出しは未実装。

### このファイルで何を作ってほしいか

- handlerからの呼び出しを受けて、repositoryを呼び出す仕組みを作ってほしい。こまかいビジネスロジックは後回しでOK。

### 参考になるファイル

services/UserService.ts

## repository

repositories/配下にあるファイル。prismaを使ってDBにアクセスする。

### このファイルで何をするのか

- prismaを使ってDBにアクセスする
- CRUD操作を実装する
- ここにビジネスロジックは書かない。ただDBの操作を関数の形で提供するだけ。

### このファイルの完成状況

- 未完成。CRUD操作の関数が未実装。
- UsewrRepository.tsにサンプルがある。

### このファイルで何を作ってほしいか

- serviceから呼び出し、情報を返したり保存したりする関数を実装してほしい。

### 参考になるファイル

repositories/UserRepository.ts
